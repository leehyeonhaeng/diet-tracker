## Day 1 — 2026.06.17

### 오늘 한 것

식단 관리 웹앱을 처음부터 끝까지 만들어서 배포까지 완료했다.

### 배경

헬스장을 잠시 못 나가는 상황에서 러닝으로 대체하면서 식단 관리가 필요해졌다. 노션에 기록하던 걸 더 편하게 쓰고 싶었고, 주변 사람들도 같이 쓸 수 있는 앱을 만들기로 했다.

**현재 상황**
- 키 180cm, 체중 85kg → 목표 75kg
- 러닝 주 3회, 총 20km / 1~2주 후 헬스장 복귀 예정
- 평일 식단 고정, 주말은 일반식

### 식단 설계 과정

처음 식단:
- 아침: 그릭요거트 100g + 냉동 블루베리 50g
- 점심: 밥 200g + 닭가슴살 200g
- 저녁(운동 후): 밥 150g + 닭가슴살 100g + 프로틴 2스쿱
- 총 칼로리: 약 1,000~1,100kcal

문제: 칼로리가 너무 낮았다. TDEE 2,500~2,600kcal 대비 -1,400kcal 적자로 대사량 저하 리스크.

**수정 식단:**
- 달걀 3개 추가 (아침 2개 + 점심 1개)
- 아몬드 40g 추가
- 프로틴 2스쿱 확정
- 최종 칼로리: 1,523kcal (러닝 기간) / 1,733kcal (헬스 복귀 후)
- 단백질 152g, 지방 43g으로 대사량 보호하면서 감량 가능한 구간 확보

### 앱 개발 과정

**v1: 클로드 채팅 내 인터랙티브 위젯**
- 로컬스토리지 기반 저장
- 기기 간 동기화 불가 → 한계 발견

**v2: 노션 연동**
- 노션 Integration 토큰 + DB ID 입력
- 저장 버튼 누르면 노션 API로 직접 저장
- 문제: 클로드 채팅 iframe에서 CORS 차단으로 노션 API 호출 불가

**v3: GitHub Pages 배포 + 노션 연동**
- HTML 파일 단독 배포
- 토큰 하드코딩 → 깃헙 secret scanning 경고 발생
- 토큰을 코드에서 제거하고 브라우저 로컬스토리지에 저장하는 방식으로 변경

**v4: 음식 자동완성 + 칼로리 자동계산**
- 기본 음식 DB 25종 내장
- 음식명 입력 시 자동완성
- g 입력 시 100g 기준으로 kcal 자동계산

**v5: 내 음식 DB**
- 사용자가 직접 음식 추가 가능 (100g 기준 영양소 입력)
- 로컬스토리지에 영구 저장
- 자동완성에서 "내 DB" 뱃지로 구분

**v6: Supabase 멀티유저 (최종)**
- 구글 OAuth 로그인
- Supabase PostgreSQL DB에 식단 저장
- RLS로 유저별 데이터 완전 격리
- 누구든 URL 접속 + 구글 로그인만 하면 바로 사용 가능

### 기술 결정 기록

| 결정 | 이유 |
|------|------|
| Vanilla JS (React 없이) | 빌드 환경 없이 GitHub Pages에 바로 배포 가능 |
| GitHub Pages | 무료, 레포에 파일 올리면 자동 배포 |
| Supabase (Firebase 아닌) | SQL 기반으로 구조 명확, RLS 설정 간단 |
| 노션 연동 제거 | 멀티유저 환경에서 노션은 개인 토큰 필요, 확장 불가 |

### 트러블슈팅

**문제 1: 깃헙 secret scanning 경고**
- 원인: HTML 파일에 노션 API 토큰 하드코딩
- 해결: 토큰을 코드에서 제거, 브라우저 로컬스토리지에 저장하는 방식으로 변경

**문제 2: 클로드 iframe CORS**
- 원인: claude.ai iframe 보안 정책으로 외부 API 요청 차단
- 해결: GitHub Pages로 독립 배포

**문제 3: Supabase UI 변경**
- 원인: 문서와 다르게 메뉴 위치가 바뀌어 있음
- 해결: Settings → API Keys 경로로 anon key 확인
